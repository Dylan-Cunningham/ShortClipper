# ShortClipper

Local app for turning timestamped clip plans into stitched short-form videos.

## App Shape

- `Frontend/` contains the Vite + React workbench.
- `Backend/` contains the local API that parses clip plans, runs ffmpeg, and manages project files.
- `outputs/` is the default export folder.
- `temp/` is used for intermediate segment files.
- `examples/` is for sample transcripts and clip plans.

## First Version

This version expects:

- a local video path, usually an iPhone `.mov` or `.mp4`
- a transcript created inside ShortClipper or uploaded from another tool
- a timestamped clip plan, usually generated by the built-in ChatGPT prompt builder
- an output folder

It cuts the requested timestamp ranges and stitches the pieces into one `.mp4` per short.

## Clip Selection Flow

1. Open ShortClipper.
2. Paste the local video path into `Video file path`.
3. Click `Create transcript`.
4. Review that the generated transcript loaded into the ChatGPT prompt builder.
5. Click `Copy ChatGPT prompt`.
6. Paste the prompt into ChatGPT.
7. Copy ChatGPT's returned JSON block.
8. Paste that JSON into ShortClipper's `Clip plan`.
9. Choose whether to append an ending CTA.
10. Click `Parse`, review the pieces, then click `Generate clips`.

Transcript files are saved in:

```text
D:\Dylan\projects\ShortClipper\outputs\transcripts
```

The app exports transcript `.txt`, `.json`, `.srt`, and `.vtt` files.

The default `.txt` transcript is now sentence-based and built from word-level timestamps. ShortClipper also writes:

- `.sentences.txt`
- `.sentences.json`
- `.words.txt`
- `.words.json`

## Ending CTA Cards

ShortClipper can append a title card to the end of every generated short.

Built-in presets:

- `READ WITH ME TOMORROW` / `1 NEPHI 2`
- `FOLLOW FOR` / `THE NEXT CHAPTER`

The default `Poster frame` style samples a still frame from the source video, turns it black-and-white, darkens it, and overlays orange/white CTA text. The older `Minimal black` style is still available.

The text, duration, style, and background frame second can be edited before rendering.

## Cut Smoothing

ShortClipper adds a little breathing room around clip ranges by default:

- Smooth continuous edit style
- 0.75 seconds before each segment
- 0 seconds after each segment
- nearby pieces within 15 seconds are merged into one continuous cut

These values can be changed in the render step.

## Local Dev

Install dependencies once:

```powershell
cd D:\Dylan\projects\ShortClipper\Backend
npm.cmd install

cd D:\Dylan\projects\ShortClipper\Frontend
npm.cmd install
```

Run the backend:

```powershell
cd D:\Dylan\projects\ShortClipper\Backend
npm.cmd run dev
```

Run the frontend:

```powershell
cd D:\Dylan\projects\ShortClipper\Frontend
npm.cmd run dev
```

Open the Vite URL, usually `http://localhost:5173`.
